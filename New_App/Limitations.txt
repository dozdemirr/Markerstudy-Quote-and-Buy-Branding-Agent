## Key Design Deviations / Notes

1. TRUST STRIP: Removed from all pages as specified.

2. PROGRESS BAR: The old separate <nav class="progress-bar"> below the header has been replaced
   with <nav class="progress-nav"> inside the header wrapper on the same row as the logo.

3. CONNECTING LINES: CSS pseudo-element lines connect progress dots. Active/done steps show
   navy lines; inactive steps show grey. This is an approximation without the exact pixel
   measurements from design screenshots.

4. YOUR-CAR.HTML SIMPLIFICATION: The legacy your-car.html included driving history questions
   (licence type, NCB, claims, convictions). In the new journey these are separate pages 
   (licence-type.html, licence-years.html, claims.html, motor-convictions.html). The 
   your-car.html now collects car-specific questions only and routes to overnight-parking.html.
   The NCB field was moved to your-cover.html.

5. ADDRESS LOOKUP: Postcode lookup is stubbed — a predefined list of addresses is shown for
   known postcodes (SW1A1AA, M11AE, B11BB, EH11YZ, LS11BA, BS11AA) and a generic list for
   all other postcodes. No real API is called.

6. VEHICLE LOOKUP: Stubbed with three demo registrations (D500NLE, AB12CDE, XY21ZZZ). Any
   other registration returns a Volkswagen Golf as a fallback.

7. OCCUPATION SEARCH: Local text-match search against a list of ~38 common occupations.
   No live API call is made.

8. DATA PERSISTENCE: All journey data is stored in sessionStorage with 'sgq_' prefix via
   the Store object. Each page guards against missing data by redirecting to index.html.

9. FONT LOADING: Google Fonts (Inter) is loaded via CDN. The app requires internet access
   for optimal typography; a sans-serif fallback is specified.

10. QUOTE CALCULATION: The quote price is calculated client-side using the existing calculateQuote()
    stub function with age, NCB, cover type, excess, claims and convictions factors.

11. ADDITIONAL DRIVERS: The additional-drivers.html page collects Yes/No only; actual additional
    driver entry flow is not implemented (consistent with legacy stub approach).
